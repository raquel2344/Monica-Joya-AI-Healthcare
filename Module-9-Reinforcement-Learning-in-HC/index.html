RL Framework Summary
Key Design Decisions

The RL scenario was built around a Markov Decision Process where the agent observes patient vitals, lab values, and treatment history to recommend antibiotic timing and fluid resuscitation. Reward signals were shaped to balance immediate stabilization against long-term survival outcomes — with hard constraints preventing the agent from recommending clinically unsafe actions.

Module Reflection

"Designing this RL scenario brought together everything from the semester — the data quality questions from Module 2, the clinical trust issues from Module 3, the ethics from Module 7, and the urgency of sepsis from Module 4. It reminded me that AI in healthcare is not a technical problem with a clinical wrapper. It is a clinical problem that technology can help solve — if we build it carefully, humbly, and with the patient at the center."
